HDFS 权限问题：Permission denied

  • 2018/5/11 0:02
    异常信息:

    Exception in thread “main” org.apache.hadoop.security.AccessControlException: Permission denied: user=Administrator, access=WRITE, inode=”/spark/global”:root:supergroup:drwxr-xr-x

    at org.apache.hadoop.hdfs.server.namenode.FSPermissionChecker.checkFsPermission(FSPermissionChecker.java:271)
    at org.apache.hadoop.hdfs.server.namenode.FSPermissionChecker.check(FSPermissionChecker.java:257)

    其实这个错误的原因很容易看出来，用户Administator在hadoop上执行写操作时被权限系统拒绝.
    详见

    1. 增加本地环境变量HADOOP_USER_NAME为hadoop的合法用户。
    2. 修改本地用户账号
    3. 修改hdfs配置 core-site.xml ，添加 dfs.permissions : false, 并修改目录权限可写入
  • 2018/5/12 23：51
    异常信息：
    Class com.mapReduce.NwordCount$TokenizerMapper not found
    找不到mapper类
    job.setJarByClass(NwordCount.class);//通过传入的class找到job的jar包
    原因：jar包的mainclass不是NWordCount，改正即可。jar包的主类应是该程序入口。
    MR默认Counter的意义
    说明：
    job.setOutputKeyClass(Text.class);
    job.setOutputValueClass(Text.class);
    两个函数设置整个job的输出，即reduce的输出。默认情况下，map的输出类型与reduce一致，若二者不一致，则需
    job.setMapOutputKeyClass(Class<?> theClass)
    job.setMapOutputValueClass(Class<?> theClass)
    两个方法来指定map的输出类型。
    2.关于输入类型的说明：reduce的输入类型即map的输出类型。map的输入类型由输入格式决定。如输入格式是
    FileInputFormat或TextInputFormat则输入KV类型为LongWriteralbe和Text
